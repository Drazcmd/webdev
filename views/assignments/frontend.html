In this assignment you will partially implement and test the frontend of your web application.

We start with our draft front-end application that contains three views: landing page with registration and login; main page with posts, followers, and status message; and the profile page. Because the dummy server requires authentication, we start this assignment by implementing the login logic. After we have logged in, we then request posts, list of followers, and the status of the logged in user from the server. Finally we implement the ability to update the status message and update the followed users list. In the next assignment we will implement adding new posts, commenting on posts, and uploading images with posts, in addition to the profile page and new user registation.

Most importantly we want to peform test driven development as we implement our webapp. Therefore before we implement anything we will first write tests for the desired functionality. In this way the desired behavior will drive our implementation and design.

Dummy Server

Please refer to the list of endpoints which is your API to a dummy data server. You have been pre-provided a user on the dummy server in addition to a test user.

Unit Testing

Write Jasmine unit tests for your JavaScript functionality. We want to test as much code as possible. Write a Karma configuration to test your site using Chrome. Try to maximize your testing code coverage.

Use Chrome as your standard supported browser. (Whereas you can use any browser you like for development, your assignment will be accessed using Chrome by the grading staff and therefore it behooves you that it works.) Host your webpage locally using the python server.

We have an Angular application and therefore want to refactor our web app into a Single Page Application. Merge together two two pages from the previous assignment into a single index.html with Angular views and routing. The structure of your web app should follow that of the RiceBookApp example where we have a top level index page, an app directory, and then subdirectories for each component such as posts or users, etc. Each subdirectory would have at least a view html file, a controller js file, and a spec js file. You also might have separate services or directives directories which contain those definitions; one factory, service, or directive per file. As inspired from an in class exercise, you should have a directive representing a post.

Remember separation of concerns and write DRY modularized code.

End-to-End Tests

Your end-to-end tests will run against your web app running on your local python server. Here are the tasks for your end-to-end test

  1. Register a new user
  2. Browse sample content, validate the number of posts
  3. Log in as your test user
  4. Create a new post and validate the post appears in the feed
  5. Update the status headline and verify the change
  6. Count the number of followed users
  7. Add the user "Follower" to the list of followed users and verify the count increases by one
  8. Remove the user "Follower" from the list of followed users and verify the count decreases by one
  9. Search for "Only One Post Like This" and verify only one post shows, and verify the author
  10. Navigate to the profile view and verify the page is loaded
  11. Update the user's email and verify
  12. Update the user's zipcode and verify
  13. Update the user's password, verify a "will not change" message
