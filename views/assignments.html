In this assignment you will create web pages to demonstrate your understanding of HTML and basic JavaScript functionality. We will create semi-functional main page and registration page stubs for our web application. The main page will consist of "cards" that contain recent blob postings or picture uploads. A "card" can be an image or a block of text.

For this assignment, do not follow unobtrusive coding style: Each page should be self-contained with an internal script tag -- no separate javascript files. Use Chrome as your standard supported browser.

Main Page Requirements
  • Use a <table> to layout a regular grid of "cards". There should be a sufficient number of cards so that scrolling down is necessary to view them all.
  • Each card should either be an image or a block of text, e.g., lorem ipsum. Have about the same number of each.
  • At least one card should have both text and an image
  • At least one card should have a link that opens a new window and navigates to an external site, e.g., http://www.rice.edu
  • Use an interval to periodically cycle through different images for each image card
  • Each image card contains a button to stop the interval, and another button to start the interval. All intervals are started by default.
  • Use a timeout (after several seconds) to inform the user to login or that they can register for a free account
  • A button to login, results in window popup when clicked with an informative message.
  • A link to register for a new account, takes the user to the registration page (see below)
Registration Page Requirements
  • A form with text fields for users's
      • desired account name first name last name address (optional) city state phone number (optional) date of birth email address password password confirmation
  • There should also be a hidden field that contains the current timestamp in millis
  • Clear button that empties all fields (resets the timestamp)
  • Submit button that performs validation on the form
    • All fields are required except those that are optional.
    • Fields should be validated for proper content, i.e., valid phone number, valid email address. [These do not need to be exhaustive]
    • leave a stub for (or implement if you like) city+state lookup
    • validate password equality with confirmation field
    • leave a stub to check if the account name is available
  • The submit button makes a GET request to the form processor page (see below) with the form contents
Form Processor Requirements
  • Extract the query parameter from the location and display the key-value pairs in a table
  • The "hidden field" timestamp should be rendered as a human readable datetime
