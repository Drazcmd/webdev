Using Windows in COMP 431 / COMP 531

The Windows operating system poses a collection of challenges for web developers. Naively everything should work out of the box. However, much of what we do in web development utilizes the command line. Additionally, compiling from source sometimes poses problems. Heroku for example requires Ruby, and we also utilize some Python -- so we have lots of resources that are desired. It has been 50-50 if Windows environments "work" or require a lot of "work".

In Windows there are a few options:

  • Windows command prompt
  • Cygwin bash
  • Git bash
  • Windows Ubuntu bash

Using the command prompt alone might work, but you might find yourself feeling like you're translating languages.

Personally I use Cygwin in Windows land. This being said, I typically have to re-engineer my solutions for every Windows machine I touch. I.e., my Cygwin setup on my desktop is different than the Cygwin setup on my laptop. This requires a great deal of dedication and persistence.

Git bash may be a good solution, but I have not spent much time with it given that I already have Cygwin.

The last item in the list is very intriguing, but is still rather new, and I don't yet have any feedback to give you on it.

Virtual Machine

A gauranteed solution is to use a virtual machine (VM) and run Linux within it. I recommend the most recent version of Ubuntu available on the linux downloads page. The instructional staff can provide some assistance in getting your VM running and configured, but you are advised to following the documentation and see if you can get it running yourself. In particular, you will likely want to have some shared drive between the VM and your Windows machine, as well as forward ports so that you can access your VM's website from your Windows browser.
